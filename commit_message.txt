feat: Implementación de mejoras completas para CordoEats

Este commit introduce una serie de mejoras significativas en el frontend y backend de la aplicación, así como optimizaciones en los workflows de CI/CD.

Frontend:
- **Branding completo:** Actualización de "DUO Previa" a "CordoEats" en `manifest.json`, `App.js` y archivos de administración.
- **UX/UI mejorado:**
    - Componentes `MenuItem` y `MenuCategories` refactorizados y movidos a `src/components`.
    - Efectos visuales de hover en `MenuItem` y gradientes en `MenuCategories`.
    - Indicador de carga visual (`LoadingSpinner`) en el menú.
- **Mobile-First Enhancements:**
    - Navegación inferior (`BottomNav`) para móviles.
    - Búsqueda inteligente con historial y sugerencias avanzadas (`SearchBar`).
    - Mejora del PWA install prompt con un botón personalizado.
    - Gestos de deslizamiento para eliminar ítems del carrito (`CartItem`).
    - Detección de ubicación para sugerir zonas (geolocalización básica).
- **Mejoras de Performance:**
    - Lazy loading de imágenes en `MenuItem`.
    - Caching inteligente de API calls para el menú.
    - Optimización del `CartContext` para persistencia en `localStorage`.
- **PWA Avanzado:**
    - Service Worker mejorado con fallback offline, background sync y manejo de notificaciones push.
    - Notificaciones de actualización de la aplicación.

Backend:
- **Refactorización de autenticación:**
    - Centralización de la configuración con `config.py` (Pydantic Settings).
    - Uso de excepciones personalizadas (`exceptions.py`) para un manejo de errores más claro.
    - Corrección de errores y mejora de la lógica en `auth.py`.
    - Integración de la nueva configuración y manejadores de excepciones en `main.py`.
- **Docker:** Creación de un `Dockerfile` básico para el backend.

CI/CD (GitHub Actions):
- **backend_ci.yml:** Actualizado a Python 3.10 y añadido caché de dependencias.
- **frontend_ci.yml:** Añadido caché de dependencias para los trabajos de linting, E2E y Lighthouse.
- **gcloud-run.yml:** Actualizados nombres de imagen/servicio, añadido paso de variables de entorno y parametrizada la región.
- **vercel-deploy.yml:** Añadida variable de entorno `REACT_APP_BACKEND_URL` para el frontend.

Este commit prepara la aplicación para un despliegue más robusto y una mejor experiencia de usuario.
